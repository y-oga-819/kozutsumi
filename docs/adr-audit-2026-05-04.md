### その他構造的気付き

- **「現状継続」型 ADR**: 0028 のように「既存パターンを続ける」判断を ADR 化するのは、他選択肢の評価記録として価値がある一方、内容が薄くなる傾向。本リポジトリでは数が少なく問題化していない。

## 全体所感

44 本中 29 本が健全、15 本が borderline、弱い ADR は 0 本という構成は、運用ルール (skill `kozutsumi-adr`) が機能していることを示している。borderline の多くは「skill §5 (Accepted 本文不変)」と「skill §1 (1 判断 = 1 ADR)」の境界事例で、ADR 化したい設計判断が深く構造化されている (0016 の Stack View / 0023 の migration diff CI / 0034〜0035 の calendar lifecycle) ほど 1 ADR に sub-decision が膨らむ傾向が見える。最優先で対処すべきは番号衝突 (0042) と ADR 0031 の broken references で、これは即座に修正可能な構造的瑕疵。次点として 0023 の改訂蓄積を split + supersede で整理し、0016 / 0035 の split を Phase 計画タイミングで検討すれば全体の規律はさらに保てる。
