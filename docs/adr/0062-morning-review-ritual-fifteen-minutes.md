# ADR 0062: 朝 15 分の棚卸し儀式 (default opt-in、自動起動しない)

- **Status**: Accepted
- **Date**: 2026-05-10
- **Related**: [Issue #234](https://github.com/y-oga-819/kozutsumi/issues/234) / [ADR-0057](./0057-redefine-moat-as-goal-driven-ai-decomposition.md) / [ADR-0058](./0058-timer-three-verbs-and-no-ai-interruption.md) / [ADR-0059](./0059-one-tap-interrupt-recording.md) / [ADR-0063](./0063-no-intervention-on-hyperfocus.md)

## Context

ADR-0057 で差別化軸の深層 (行動データ蓄積による分解精度個人最適化) を定義したが、ADR-0058 で「timer 中は AI 黙る」原則を確定したため、AI が能動的に語れる場を timer 文脈外に確保する必要がある。

調査 4 本横断で「朝の Task Recovery Score」(Whoop 型、#234 Comment 4) は「ユーザー定着の単一最強機能」と評価され、Bullet Journal の Migration / タスクシュート朝レビューも同型の儀式として支持されている。

加えて以下の付帯事項を吸収する場が必要:

- ADR-0059 (1-tap 割り込み記録) で事前分類しないと決めたため、振り返りでの分類が必要
- ADR-0063 (過集中介入なし) で stop し忘れて長時間記録が残るケースの事後補正が必要
- 想定ユーザー特性 ② 興味の有無で進捗が極端に変動、への処方として stack 並び替えを毎朝実行できる場が必要

「差し込まれるとやる気を削ぐ」体感 (#234 議論) を尊重するため、自動起動 (起動時強制表示) は採用しない。

## Decision

1 日の最初に開く専用画面として「朝 15 分の棚卸し」儀式を作る:

1. **昨日の summary** — 完了 / 中断 / 割り込み件数 + 行動データの示唆 (Whoop Recovery Score 風)
2. **今日の stack 確認** — 重い順 / 軽い順並び替え
3. **「今日の意図」を 1 つ選ぶ** — その日のフォーカスを 1 つだけ宣言
4. **過集中の事後補正** — 長時間記録が残ったタスクの実作業時間補正 + 「時間に見合う成果が出たか」の妥当性振り返り (具体 schema は別途決定)

15 分で完結するキャップ。週次振り返り / 先送り判断 (Anki Leech 型) は別セッションに分離する。

起動方式は **default opt-in** (ユーザーが明示的に開く)。**自動起動 (起動時に強制表示) は採用しない**。

## Consequences

### 肯定的影響

- 「秘書らしさ」を timer 中に出さず、棚卸し画面に集中させる二画面構成が成立する (画面レイアウトの詳細は機能要件確定後に別命題として整理)
- 過集中の事後補正がここに集約される。timer 中は介入しない (ADR-0063) が、振り返りで補正する非対称構造が成立する
- ADR-0059 で事前分類しなかった割り込みデータの分類 / タスク化判断もここで行える
- default opt-in で「差し込まれるとやる気を削ぐ」体感を尊重。Davis "morally neutral" / 借金玉「自分を変えるな道具に頼れ」と整合
- 「ユーザー定着の単一最強機能」(Comment 4) 評価のとおり、長期定着への寄与が期待できる

### 否定的影響・トレードオフ

- opt-in なので習慣化は user の自律に依存する。儀式として機能しない可能性あり
- 新規ビューの実装 + 新 entity (daily-review 相当) が必要。実装コスト中
- 過集中事後補正の schema (実作業時間と zone time の二値、見積もり補正への入力扱い、auto-pause opt-in 有無) は別途決定が必要

## Alternatives considered

- **起動時自動表示 (儀式化を強制)**: 「差し込まれるとやる気を削ぐ」体感を尊重。Habitica / Duolingo 型強制で挫折するパターンと同じ。不採用 (ただし dogfooding で「自動でも嫌じゃない」感触が出たら再検討余地あり)
- **c1 軽量版 (今日の stack 確認 + 今日の意図のみ)**: 過集中事後補正と行動データ示唆を吸収できない。不採用
- **c3 重量版 (週次 / 月次振り返り + 先送り判断を含む)**: 15 分キャップを超える。週次は別セッションに分離する判断と整合せず。不採用

## Notes

- 過集中事後補正 schema の詳細 (実作業時間と zone time の二値で持つか単一カラム上書きか / 補正値を見積もり補正エンジンに入力するか除外か / auto-pause を opt-in で用意するか) は M-α 期間の dogfooding 後に確定する
- 画面レイアウト / 体験設計 (stack view との二画面構成、導線、デフォルト起動画面、相互の遷移パターン) は機能要件 (ADR-0061 / 0062 / 0064) が出揃ってから別命題として整理する。本 ADR では扱わない
- 将来見直す条件: dogfooding で「opt-in だと開かない」「自動起動でも嫌じゃない」が確認できたら起動方式を再検討
