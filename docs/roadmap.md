# ロードマップ

## フェーズ計画

### Phase 1: コアUI（AIなし・カレンダーなし） ✅

手動でイベントとタスクを入力し、イベント駆動スタックの体験を検証。Vercel へのデプロイまで完了し、日常利用できる状態に到達した。

実装は GitHub Issue で管理（Milestone `Phase1：コアUI` 配下、Epic は #22）。設計判断は [docs/adr/](./adr/) を参照。

検証仮説と成功基準:
- スタックのトップから1個ずつ消化が7日間継続できるか
- 割り込みpush/popで元タスクに戻れた率80%以上
- タイムラインバーが空き時間の判断に役立つか
- 見積もり時間と実作業時間の記録が自然にできるか

### Phase 2: Google Calendar連携

カレンダーからイベントを自動取得。依存関係を手動設定可能に。

### Phase 3: AIサジェスト + 見積もり補正

- タスク追加時の自動分解（承認ステップなし、直接スタック挿入）
- 依存関係の推論
- 見積もり補正エンジン（過去実績からの自動補正）
- 行動ログの蓄積開始

### Phase 4: 二段階提案 + 行動パターン分析

- スコアリングモデル（ボトルネック度、実質残り日数、エネルギー適合度を含む）
- LLM妥当性検証
- 朝のスタック自動提案
- 暗黙フィードバックによる重み更新
- 着手パターン・回避パターン・エネルギー変動の分析
- 苦手タスクの原因推定と対処提案


## 将来構想: 横展開

タスク管理で蓄積された個人コンテキスト（思考パターン、判断基準、文体の好み、プロジェクトの文脈）は、他のドメインでも活用できる。

- **ドキュメント生成支援:** 「この人はSLOの文書を書くとき、まずユースケースから入る」という構成の好みを反映
- **MTGアジェンダ自動生成:** 関連タスクの進捗からアジェンダ項目を提案
- **週報の下書き:** 完了タスクと行動ログから自動生成
- **メールドラフト:** 文体と判断基準を反映した返信案

全て同じ個人コンテキストの上に載る。タスク管理は最初のアプリケーションに過ぎない。
