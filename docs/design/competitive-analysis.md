# 競合分析

## Sunsama（$20/月）

毎朝の「プランニングリチュアル」で各種ツールからタスクを引っ張り、手動で時間ブロックに配置するデイリープランナー。

強み: 意図的な計画習慣の構造化。20以上のツール連携。見積もり時間 vs 実時間のトラッキング。

弱み: AIによる自動スケジューリングがない。計画は全て手動。毎日儀式をやらないとただの高額タスクリスト。

## Motion（$29/月）

AI完全自動スケジューリング。タスクに優先度・締切・所要時間を入力すると、AIが最適なスロットに自動配置。予定が変わると数秒で再スケジュール。

強み: 計画の手間がほぼゼロ。動的な再スケジューリング。

弱み: ユーザーが制御を失う。「今これをやるべき理由」が見えない。タスクを「優先度・締切・所要時間」の属性セットとして扱い、個人の行動パターンは考慮しない。

## kozutsumiの差別化

Motionとの機能レベルの差別化はモートにならない。フィードバック学習もUI思想もMotionが資金力で追随できる。

kozutsumiの本質的な差別化は「行動パターン分析の深さ」にある。

Motionは属性ベースでスケジュールする。タスクの優先度、締切、所要時間という静的な属性から最適配置を計算する。

kozutsumiは行動ベースでスケジュールする。着手パターン、中断パターン、回避パターン、エネルギー変動、タスク種類ごとの見積もり誤差、時間帯×タスク種類のクロスパターンという動的な行動データから最適配置を計算する。

この違いは蓄積データの構造が根本的に異なることに起因する。Motionが「機能を真似する」だけでは追いつけない。なぜなら行動データを取得するためのデータ設計思想が違うから。

| 観点 | Sunsama | Motion | kozutsumi |
|------|---------|--------|-----------|
| 計画の主体 | 人間（手動） | AI（完全自動） | AI提案 → 自動挿入 |
| スケジューリング基盤 | なし | 属性ベース | 行動パターンベース |
| 見積もり | ユーザー入力のまま | ユーザー入力のまま | 過去実績から自動補正 |
| フィードバック学習 | なし | なし | 暗黙的に蓄積・学習 |
| 苦手タスクへの対処 | なし | 締切で強制配置 | パターン分析→分解→配置最適化 |
| 割り込み対応 | 手動で再配置 | 自動再スケジュール | スタックpush/pop+パターン記録 |
| 使い続ける効果 | 変わらない | 変わらない | 精度が上がり続ける |
