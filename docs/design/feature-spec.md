# 機能仕様

UI機能の詳細と成功指標をまとめる。アーキテクチャ上の機構（ [architecture.md](./architecture.md) ）を、どんな画面・操作で見せるか。

## 1. 機能設計

### 1.1 デイタイムライン

1日の時間構造を横バーで可視化。

- 9:00〜18:00を基本、予定がある場合は動的に伸びる
- イベントはプロジェクト色のブロック、各ブロックに所要時間
- 空き時間ブロックに所要時間
- 現在時刻の緑マーカー、現在の空き時間をハイライト

### 1.2 イベントカード一覧

タイムラインバーの下にイベントを時系列のカードで表示。

- 時刻、所要時間、タイトル
- 添付資料ありの場合はドキュメントアイコン
- 会議URLありの場合はビデオアイコン
- NEXTイベントには「Meetに参加」ボタンを直接配置
- カードタップで詳細パネル

### 1.3 タスクスタック

スタックの最上位を大きなカードで表示し、「今これをやる」を明確にする。

- 最上位: プロジェクト色、タイトル、補正後見積もり時間、依存イベント
- 2番目以降: コンパクトな1行
- ドラッグ&ドロップで並べ替え（モバイルタッチ対応）
- 完了ボタンで実作業時間を自動記録してdoneへ移動
- 割り込みpushボタン: 現在タスクを中断して新タスクをトップへ（⚡マーク）
- カードタップでMarkdown詳細パネル

### 1.4 Tree View（履歴ビュー）

git log tree風のUIで過去の活動を振り返る。

- 縦軸が時間、横軸がプロジェクト
- 各プロジェクトが色付きの縦線（ブランチ）
- 完了タスクがノード、割り込みタスクは別マーク
- 日付区切り


## 2. 成功指標（KPI）

### 2.1 日次

- **スタック消化率:** 今日のスタックで完了した割合。目標: 70%以上
- **見積もり精度:** 補正後見積もりと実作業時間の乖離。目標: ±20%以内が80%
- **トップ開始率:** スタックのトップから作業を開始した割合。目標: 80%以上

### 2.2 週次

- **提案承認率:** AI提案を大幅修正なしで使った割合。Phase 4開始後に50%→3ヶ月で70%
- **プロジェクトバランス:** 本業 vs それ以外の消化時間比率
- **割り込み率:** 全タスクに対する割り込みタスクの割合
- **回避タスク検出数:** 3日以上連続で後回しにされたタスクの数

### 2.3 継続

- **連続利用日数:** 目標: 30日継続
- **放棄パターン:** 使わなくなった日の前後の状況分析
- **見積もり精度の改善率:** 月ごとの乖離率の推移
